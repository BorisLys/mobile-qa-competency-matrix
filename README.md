# Mobile QA Competency Matrix

## Общие правила

- **Ожидания каждого следующего грейда включают ожидания предыдущих.**  
  Например, *Middle+* должен соответствовать требованиям **Junior + Middle**  
  (минимум **32 балла** по навыкам этих уровней).

- **Оценка 32 должна подтверждаться ссылками**  
  на выполненную работу в рамках навыка.

- **Для перехода на Senior необходимо овнерство проекта гильдии.**  
  Примеры:
  - овнерство key result;
  - ответственность за тестовую модель;
  - ответственность за онбординг и процессы.
 
- Разделы, помеченные звёздочкой (*), **не являются обязательными** для соответствия грейду.  
  Они не блокируют рост, но **помогают набрать дополнительные баллы** и усиливают профиль специалиста.

---

## Пороговые значения по грейдам

Для соответствия грейду необходимо набрать минимальное количество баллов
по всем учитываемым навыкам.

- **Junior** — от **208** баллов  
- **Middle** — от **432** баллов  
- **Middle+** — от **1024** баллов  
- **Senior** — от **1632** баллов  

Баллы считаются **накопительно** с учётом требований предыдущих грейдов.

---

## Оценки за пределами матрицы

- **64** — системная работа на уровне проекта с результатом.  
  Делегирование через других QA, самостоятельное решение неделегируемых задач, проактивность.

- **128** — влияние на уровне гильдии или нескольких команд.  
  Выстроенная система, масштабируемая или автономная.

- **256** — экспертиза уровня индустрии.  
  Глубокое владение темой, сильное состояние проекта, команда — отражение экспертизы.

**8 баллов**

- State Transition
- Pairwise
